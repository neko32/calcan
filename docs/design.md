# calcan 設計仕様書

線形代数計算ドリルをフラッシュカード形式で学習できるWebアプリケーション「calcan（カルカン）」の設計仕様書まる。

---

## 🎯 システムの目的
本システムは、数学的計算能力の向上を目的とし、**電卓で解けるレベルの線形代数計算問題**をフラッシュカード形式で出題するまる。
電卓による計算を前提とした数値・行列サイズに調整された全20問の良問を提供し、視覚的な美しさとスムーズな操作性で学習をサポートするまる。

---

## 🛠 技術スタック
- **フロントエンドライブラリ**: React (TypeScript)
- **ビルドツール**: Vite
- **数式レンダリング**: KaTeX
- **アイコン**: Lucide-React
- **テストフレームワーク**: Vitest + JSDOM + v8 Coverage
- **デプロイ・CI**: GitHub Actions (リリースビルドおよびパッケージング)

---

## 🎨 UI/UX・デザインシステム
「カルカン (猫缶)」のダジャレにちなみ、キュートかつプレミアムな猫モチーフのデザインを採用しているまる。

- **カラーパレット**:
  - メインブランドカラー: ロイヤルパープル (`#5b287a`) ➔ カルカンのパッケージカラー
  - サブカラー: ソフトピンク (`#ff7597`) ➔ 猫の鼻や肉球をイメージしたキュートなピンク
  - アクセントカラー: ゴールド (`#ffb703`) ➔ キャットフードや王冠を連想させるイエロー
  - 背景色: クリームホワイト (`#fffbf5`) ➔ 目に優しく温かみのあるクリーム地
  - ダークモード用背景: ディープパープル (`#12061a`) ➔ 深みのあるスタイリッシュなダークテーマ
- **特徴的なデザイン要素**:
  - **猫耳カード**: 問題カードの上部にCSSで猫の耳 (`.ear`) を立体的に装飾
  - **3Dフリップアニメーション**: カードをクリックするか、`Space` / `Enter` キーを押したときに滑らかにカードが回転する3Dエフェクト

## 🧩 コンポーネント設計
アプリケーションは以下の5つの主要モジュールから構成されているまる。

1. **`App.tsx`**:
   - アプリ全体の画面状態（`start` / `play` / `result`）の遷移をコントロール。
   - 選択されたコースに応じて、問題データをフィッシャー–イェーツのアルゴリズムでシャッフル。
   - キーボードの `Space` / `Enter` キーイベントをグローバルに監視し、カードの反転や「次へ」のアクションをトリガー。
   - 解答表示中に `Backspace` キーを押すことで、元の問題表示画面へカードを戻す（裏返す）キーボードショートカットを実装。
2. **`MathText.tsx`**:
   - 与えられたテキストからインライン数式 (`$...$`) およびブロック数式 (`$$...$$`) を正規表現でパース。
   - 分割されたパーツごとに `katex` レンダラーを呼び出し、高速かつ正確にLaTeX数式をHTMLに変換。
   - テキスト改行コードを検出し、自動的に `<br />` へと展開して改行レイアウトを崩さない仕組み。
3. **`StartScreen.tsx`**:
   - 「フルコース (20問)」「基礎ドリル (10問)」「応用ドリル (10問)」のコース選択画面。
   - 直感的でモダンなカード型選択肢と、動きのあるホバーエフェクト。
4. **`CardScreen.tsx`**:
   - フラッシュカード本体。
   - 表（問題面）には「カテゴリ」「問題ID」「KaTeXによる問題式」を表示。
   - 裏（解答面）には「正解の数式」と「詳細な途中計算・解説ヒント」をスクロール可能エリアに表示。
5. **`ResultScreen.tsx`**:
   - 全問クリアしたユーザーをお祝いする画面。
   - 学習レポート（挑戦コース、問題数、完了バッジ）を表示。
   - チームのバックエンドエンジニア「コバトン」からの応援メッセージを表示。

## 🧪 テスト・品質保証方針
- **カバレッジ目標**: **90%以上** の命令・分岐・関数カバレッジを保証する。
- **テストの実行内容**:
  - 各コンポーネントがPropsに応じて正しいHTMLをレンダリングすることの検証。
  - `MathText` のLaTeXテキストのパース正確性の検証。
  - カードクリック時の状態反転および画面遷移の統合シミュレーション。
  - `Space` と `Enter` キーによるインタラクションの検証。
  - 20問のドリルデータのID整合性、各フィールドの型の厳密なチェック。
- KaTeXの描画処理はJSDOM環境における依存エラーを防ぐため、テスト用セットアップファイル（`vitest.setup.ts`）内でモック化されているまる。

## 🐳 Dockerコンテナ対応
ローカルでDockerコンテナを立ち上げてアプリを起動するための環境を構築しているまる。

- **Dockerfile**:
  - マルチステージビルドを採用。
  - 第1ステージ（`build`）: `node:24-alpine` で依存関係の解決と `npm run build` によるビルドを実施。
  - 第2ステージ: `nginx:stable-alpine` にビルド成果物（`dist`）をコピーし、軽量かつ高速な静的配信サーバーとして提供。
- **起動スクリプト**:
  - `scripts_local/run_docker.ps1` (PowerShell) および `run_docker.sh` (Bash) を実行することで、Dockerイメージのビルド、既存の `calcan` コンテナの停止・削除、およびポート `5273` でのコンテナ起動が自動で行われるまる。
